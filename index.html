ply
Light-Weight Dynamic Tracing for Linux

Highlights

Syntax

The syntax is modeled on stolen from Dtrace, which in turn takes its inspiration from awk. Thus, people who have written an awk script should be able to effortlessly write ply scripts. Variables are declared and their types inferred automatically, making iterating on scripts fast and easy.

Here is a very simple example that counts the number of calls to open(2): 

kprobe:SyS_open
{
    $opens.count()
}

Light-Weight

Designed with embedded systems in mind. Written in C, all ply needs to run is libc and a modern Linux kernel with eBPF support. No LLVM, no python.

Efficient

All data gathering and aggregation is done in the kernel using eBPF programs that are JIT compiled to native instructions on most common architectures. This means that ply runs with very low overhead, allowing it to probe even the hottest code paths.

Try It

Download the source from Github. ply is known to work with Ubuntu 15.10, but should build and run equally well on any linux system with a kernel of version 4.2 or later.

Be sure to read the manual for a more complete documentation of the syntax, built-in variables and functions and example scripts.
